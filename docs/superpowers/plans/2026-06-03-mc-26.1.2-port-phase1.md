# MC 26.1.2 (NeoForge) Port — Phase 1: Server-Side Runtime Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Migrate the entire server-side runtime (~66 of 72 files) from Forge 1.20.1 to NeoForge 26.1.2, in dependency order with a green build at every step, ending with an in-game-verified Steve that spawns, responds to `/steve`, runs a basic action, and reaches Claude.

**Architecture:** The legacy source is staged under `port-staging/`. We reintroduce it bottom-up: MC-free pure-Java packages first (move unchanged), then the MC/Forge-coupled files migrated to NeoForge APIs. Migration of MC-internal APIs is **compile-driven** — we compile against the real decompiled NeoForge 26.1.2 sources already on disk and resolve exact symbols from the compiler + local sources + NeoForge docs, rather than guessing package paths.

**Tech Stack:** NeoForge 26.1.2.71, Java 25, ModDevGradle 2.0.141. Verified framework mappings: `DeferredRegister`/`DeferredRegister.Entities` + `DeferredHolder` (replaces `RegistryObject`); `ModConfigSpec` + `ModContainer#registerConfig` (replaces `ForgeConfigSpec`/`ModLoadingContext`); `@EventBusSubscriber` + `NeoForge.EVENT_BUS` game bus + mod bus from constructor (replaces `@Mod.EventBusSubscriber`/`MinecraftForge.EVENT_BUS`).

**Build command (this machine):** `./gradlew build` — system JDK 25 is the default, no `JAVA_HOME` needed.

## STAGE 1b — MC-utility layer (roadmap)

**Files to migrate (move from `port-staging/`, fix to compile):**
- `memory/` (3): `SteveMemory` (NBT `CompoundTag` save/load), `WorldKnowledge` (`BlockPos`, `AABB`, level scans), `StructureRegistry`.
- `execution/` pure-Java parts (interceptors, `AgentStateMachine`, `ActionContext`, `InterceptorChain`, `CodeExecutionEngine` [GraalVM], `SteveAPI` [MC: `BlockPos`/`Entity`/`BlockState`/`Vec3`/`AABB`]).
- `event/` bus + event data classes (5, pure Java): `EventBus`, `SimpleEventBus`, `StateTransitionEvent`, `ActionStartedEvent`, `ActionCompletedEvent`. (NOT `ServerEventHandler` — that's 1d.)
- `llm/` remainder: `PromptBuilder` (MC `BlockPos`/`ItemStack`), `ResponseParser`, `TaskPlanner`, `OpenAIClient`, `GeminiClient`, `GroqClient`.
- `util/ActionUtils`, `structure/` (3: `BlockPlacement`, `StructureGenerators`, `StructureTemplateLoader`).

**Known migration notes:** NBT API (`CompoundTag`) is largely stable; watch for `putInt`/`getInt` signature and `ListTag` changes. Item handling may touch **data components** (1.20.5+) — `ItemStack` NBT helpers changed. Resolve exact symbols by compiling against local NeoForge sources at `build/moddev/artifacts/`.

**Methodology per file:** `git mv` back → `./gradlew compileJava` → fix `net.minecraftforge.*`→`net.neoforged.*` and changed `net.minecraft.*` calls using the compiler error + local source grep → commit per package. **Gate:** stage compiles green.

**Move the matching tests** (`memory/WorldKnowledgeTest`, `structure/StructureGeneratorsTest`, `llm/TaskPlannerTest`) back with their targets.

## STAGE 1d — Entry + wiring + in-game gate (roadmap)

**Files:** `config/SteveConfig`, `entity/SteveEntity`, `entity/SteveManager`, `command/SteveCommands`, `event/ServerEventHandler`, and expand `SteveMod` from the shell into the full entry.

**Verified framework migrations:**
- **SteveConfig:** `ForgeConfigSpec*` → `ModConfigSpec*` (package `net.neoforged.neoforge.common.ModConfigSpec`). Builder/`ConfigValue`/`IntValue`/`DoubleValue`/`BooleanValue` are nested types. Keep the `[anthropic]`/`[openai]`/`[ai]`/`[behavior]` structure and Claude-default values intact.
- **SteveMod entry:** entity registration via `DeferredRegister.create(BuiltInRegistries.ENTITY_TYPE, MODID)` (or `DeferredRegister.Entities`), `RegistryObject` → `DeferredHolder`/`Supplier`. Register the DR to the **mod bus** (constructor param). Register config via the injected `ModContainer#registerConfig(ModConfig.Type.COMMON, SteveConfig.SPEC)`. Replace `FMLJavaModLoadingContext.get().getModEventBus()` with the constructor's `IEventBus`. Attribute creation listener (`EntityAttributeCreationEvent`) and command listener (`RegisterCommandsEvent`) on the appropriate bus. Drop the client-only `SteveGUI` registration here (moves to Phase 2).
- **ServerEventHandler:** `@Mod.EventBusSubscriber` → `@EventBusSubscriber(modid = SteveMod.MODID)`; `@SubscribeEvent` from `net.neoforged.bus.api.SubscribeEvent`; `PlayerEvent.PlayerLoggedInEvent/PlayerLoggedOutEvent` from the NeoForge event package; register on `NeoForge.EVENT_BUS` (game bus).
- **SteveEntity:** `PathfinderMob`, `createMobAttributes()`/`AttributeSupplier`, AI `Goal` classes, `SynchedEntityData`/`EntityDataAccessor` (the `defineSynchedData` signature changed in 1.20.5 — now takes a `SynchedEntityData.Builder`), NBT `addAdditionalSaveData`/`readAdditionalSaveData`. Verify each against local NeoForge sources.
- **SteveCommands:** Brigadier + `Commands.literal/argument` — largely stable; fix any `CommandSourceStack`/registration signature deltas.

**Gate (Phase 1 exit):**
- `./gradlew build` green.
- User runs `./gradlew runClient`; in-game: a Steve spawns (on player login / via command), `/steve <command>` is accepted, a basic action executes, and the log shows a Claude request/response (with a configured `[anthropic].apiKey`) — or a clean auth error if unconfigured.
