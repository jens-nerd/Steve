## Known MC 26.1 client-API deltas (resolve exact symbols via local sources / javap)
- **Entity render-state rewrite (ClientSetup):** `HumanoidMobRenderer`/`PlayerModel` are now parameterized by a `‚Ä¶RenderState` (1.21.5+). `PlayerModel` is no longer generic over the entity. Approach: register via `event.registerEntityRenderer(SteveMod.STEVE_ENTITY.get(), SteveRenderer::new)` where `SteveRenderer` extends the appropriate humanoid renderer with the correct render-state generic, overriding the texture accessor (now keyed off the render state) to return the player skin `Identifier`. If the full humanoid render-state wiring proves large, a correct minimal fallback is acceptable (a renderer that produces a valid `EntityRenderState` so the client doesn't crash) ‚Äî note it for polish. Resolve generics with `javap -p` on `build/moddev/artifacts/minecraft-patched-26.1.2.71.jar`.
- `ResourceLocation` ‚Üí `Identifier` (`net.minecraft.resources.Identifier`): `new ResourceLocation("minecraft", path)` ‚Üí `Identifier.withDefaultNamespace(path)` or `Identifier.fromNamespaceAndPath(ns, path)`.
- `GuiGraphics`: `fill`/`fillGradient`/`drawString`/`enableScissor`/`disableScissor` largely survive ‚Äî fix any `drawString` signature delta from the compiler. The manual `RenderSystem.enableBlend()/setShaderColor()/blendFuncSeparate(GlStateManager‚Ä¶)/disableBlend()` block is no longer needed (GuiGraphics manages blending) ‚Äî remove it; `GlStateManager` source/dest-factor constants likely no longer apply.
- `EditBox`: `tick()` no longer exists (remove `SteveGUI.tick()`'s `inputBox.tick()` call / the method if unused). `render(GuiGraphics, int, int, float)` partial-tick arg: `mc.getFrameTime()` ‚Üí `mc.getDeltaTracker().getGameTimeDeltaPartialTick(false)` (verify).
- `mc.player.connection.sendCommand(String)` ‚Äî verify it still exists (ClientPacketListener).
